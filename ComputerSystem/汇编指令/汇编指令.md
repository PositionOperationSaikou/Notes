数据传送指令
MOV 把数据从源位置传送到目标位置, 不做任何更改.

x86-64增加了限制mov指令不能两个操作数都指向内存位置.
即将一个数从内存位置拷贝到另外一个内存位置, 不能一条指令完成,
必须先拷贝到寄存器, 然后再从寄存器拷贝到内存.

movl会将高位四个字节设置为0, 即任何为寄存器生成32位
值的指令都会把该寄存器的高位部分设置成0.

movq源位置是立即数只能是4字节, 剩下的部分将这个4字节立即数符号扩展得到64位的值,
(即如果是补码实现, 正数前面补充0, 负数需要将这个值转化位64位的相同值(按位取反+1)).
源位置是寄存器或则内存则会完整拷贝64位立即数.

movabsq指令能够以任意64位立即数作位为源操作数, 但只能以寄存器为目的.
这意味着如果需要将64位立即数拷贝到内存, 则需要使用两条指令先movabsq拷贝到寄存器, 再movq拷贝到内存.
这样做的目的是避免过长的指令出现,比如将一个64位立即数拷贝到内存, 指令需要包含这个64位立即数以及计算内存偏移量、比例变地址
等二进制码, 则指令的长度可能是10多个字节.过长的指令会降低cpu的指令缓存命中率, 并且如果都使用movq拷贝64位立即数,
则意味原本使用32位立即数的场景都会扩展64位立即数. 权衡后再部分少见的情况下使用将指令扩展到两条是更好的做法.

条件码
描述最近的算数或逻辑操作的属性.

CF: 进位/借位标志, 最近运算溢出宽度（即需要当前最高位高一位发生进位或则借位）才会值为1
ZF: 零标志, 最近的操作得出的结果为0
SF: 符号标志. 最近的操作得到的结果为负数.
OF: 溢出标志. 最近的操作导致一个补码溢出.

leaq指令不改便任何条件码,  因为它是用来计算地址的.
除此之外图3.10所有的操作都会设置条件码. 
逻辑操作设置进位标志和溢出标志为0. 
位移操作进位标志设置为最后一个被移出的位, 溢出标志设置为0. 
INC和DEC会设置溢出标志和零标志, 但不改变进位标志.


cmp S, D类指令有cmpb、cmpw、 cmpl、cmpq. 行为类似SUB S, D, 
唯一的不同在于它不保存结果, 只通过计算后条件码状态完成比较, 分析如下 

相等b=a, b-a=0, zf=1

无符号
只需要判断cf, b < a则 b-a发生借位cf=1否则为0; 

有符号
cmpq a,b  如果b>a,  则b-a>0， 
情况1 a>0, b >0 则sf=0, of=0;  
情况2 b<0, a <0(符号位抵消, b的后面部分大于a结果是正数不溢出)， sf = 0, of = 0. 
情况3 b>0, a<0（a符号位1，b为0 
此时结果不溢出b(0001 - 1100)= b(0001 + 0100) = b(0101)此时of=0, sf=0
溢出 b(0111 - 1000) = b(0111 + 1000) = b(1111) 此时of=1 sf=1, 即溢出后计算结果符号发生反转.


test S, D类指令有testq、 testw、 testl、 testq. 行为类似and s, d.
唯一的不同在于它不保存结果, 只通过计算后条件码状态完成判断.
cf和of置0, 当计算结果为0是zf=1, 计算结果符号位为1sf=1.
用法是S需要检测的值, D掩码用来标记需要检测的位.
完成它只需要一个cpu时钟周期(更快更无负担),
所以经常用于替换cmp和and检测一个值是否为0 test b(0000), 
b(0000) 或则是否为负 test b(1000), b(1000).

它可以直接跳转, 即跳转地址直接作位指令编码一部分. 直接跳转的写法是指令后面跟上要跳转的标号.
例子 
jmp lable1
...... //省略中间的代码
lable1:
...... //省略后面的代码

也可以间接跳转即指令后面带的是一个寄存器或则内存位置, 它使用寄存器或内存中保存的值作为跳转目标.
间接跳转写法: 跳转指令 *后带上一个操作数指示符.
例子
jmp *%rax 使用rax值作位跳转目的地址 或 jmp *(%rax)使用rax值指向的内存地址中的值作位跳转目标.

其他指令含义都是满足某个条件即跳转, 指令判断原理和命名规则跟set指令类似. 它们只能使用直接跳转.

其一因为cpu的流水线效率这是主要原因, 现代cpu底层对于指令其实是并行执行, cpu执行到需要条件判断的指令时先进行分支预测
, 即直接把条件判断成立后跳转的指令读出来. 如果条件成立那么节省了cpu时间, 如果不成立只要丢弃读到的指令.
但如果跳转的地址不是一个固定的值而是保存在寄存器或则内存中的值则cpu在这之前没办法知道要跳转的目标.  

其二因为指令长度有限, 如果要使用间接跳转意味着必须要额外的位表示后面的部分是一个地址引用需要从它里面把值取出来, 
早期的cpu或是精简指令集这意味需要增加指令的长度或牺牲其他功能.

跳转指令的编码
汇编代码中直接跳转写法是跳转指令后面跟上一个标号.
汇编器和后来的链接器会产生跳转目标的适当编码, 这其中一共有两种方式
其一绝对地址, 用四个字节指定跳转目标.
其二偏移地址, 用1字节或2字节作位偏移量, 当前指令后面一条指令的地址作位基址. 目标地址=基址+偏移量. 
基址不选择当前地址的原因是在早期cpu上常常将更新程序计数器作位指令执行的开端.

大多数时候都会使用偏移地址的方式, 因为它更灵活编译器和链接器可以根据最节省空间的方式. 而且哪怕它变了(动态链接/加载), 只要内部偏移距离不变程序照样能跑.
只有在调用函数或是距离当前指令很远的固定系统调用的时候才会使用绝对地址, 因为当函数地址距离当前指令非常远的情况只有使用绝对地址才能准确到达.
汇编器或链接器会自己计算如果目标地址距离当前指令较近, 则使用偏移地址节省空间.
反之使用偏移地址.
